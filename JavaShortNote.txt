
What is Java ?
    Java is high-level,object-orientend programming language developed by Sun 
Microsystems(now owed by Oracle) in 1995.
    it is widely used for desktop,web,and mobile applications.

Key features:
    Platform Independent -> Write Once, Run Anywhere (WORA).
    Object-Orientend -> Based on classes and object.
    Multithreaded -> Supports concurrent programming.
    Automatic Memory Management (via Garbage Collector).

Java Development Enviroment:
    JVM (Java Virtual MAchine)
            JVM is the engine that runs java programs.
            it converts Java bytecode(compiled code) into machine code(understandable by the OS).
    JRE (Java Runtime Enviroment)
            JRE = JVM + Libraries + other components
            Provides Libraries and runtime needed to run java programs.
    JDK (Java Development Kit)
            Main toolkit for java development.
            includes compiled and development tools.

Thread ?
    Thread is a lightweight process that can allow concurrent execution of program.
    it is used to perform multiple tasks simultaneously.
Multithreading ?
    Execution of multiple threads simultaneously within a single program.

    Thread Life Cycle
        New -> Runnable -> Running -> Blocked/Waiting -> Terminated

        sleep -> Thread.sleep() method is used to pause the execution of thread for specified time.
        join -> Thread.join() method is used to wait for a thread to complete its execution before proceeding with the next thread.
        yeild -> Thread.yield() method is used to pause the execution of current thread and allow other thread to execute.
        start -> Thread.start() method is used to start the execution of thread.

Garbage Collector
    Garbage Collector is a automatic memory management process that is used to collect unreferenced objects or unused memory from the heap.
    1.Nulling object reference
    2.reassigning object reference
    3.creating anonymous object
    4.creating object inside the method
    System.gc() -> Suggests the JVM to perform garbage collection, but it does not guarantee that it will happen immediately.

Collection Framework ? JCF
    Collection is ready-made architecture 
    Collection framework is a set of classes and interfaces that provide data structure and algorithms to store and manipulate data in java.
    java.util package provides collection framework.
    list -> ArrayList, LinkedList, Vector
    queue -> PriorityQueue,Deque, ArrayDeque
    set -> HashSet, LinkedHashSet, TreeSet

        add() -> adds an element to the collection.
        remove() -> removes an element from the collection.
        contains() -> checks if an element is present in the collection.
        size() -> returns the number of elements in the collection.
        isEmpty() -> checks if the collection is empty.
        clear() -> removes all elements from the collection.

list ?
    List is an ordered collection that allows duplicate elements and maintain the insertion order.
ArrayList
    ArrayList uses a dynamic array
    It allows duplicate elements and maintains insertion order.
    Used for fast data retrieval (accessing elements)
LinkedList
    LinkedList uses a doubly linked list.
    it is not synchronized and allows sequential access to elements.
    Efficient for insertion and deletion operations
Vector
    Vector uses a dynamic array.
    it is synchronized and allows duplicate elements and maintains insertion order.
    Used for thread-safe operations but slower than ArrayList.
    default capacity 10
Stack
    Stack is a last-in-first-out (LIFO) data structure.
    it is synchronized and allows duplicate elements.
    Used for storing temporary data and performing backtracking operations.
        push() -> adds an element to the top of the stack.
        pop() -> removes and returns the top element of the stack.
        peek() -> returns the top element of the stack without removing it.
        search() -> returns the 1-based position of the first occurrence of an element in the stack, or -1 if not found.

Set
    Set is a collection that does not allow duplicate elements and does not maintain insertion order.
HashSet
    HashSet uses a hash table for storage.
LinkedHashSet
    LinkedHashSet uses a hash table and a linked list for storage.
TreeSet
    TreeSet uses a tree for storage.

Queue
    Queue is a collection that follows the first-in-first-out (FIFO) principle.

Curser
    Curser is an object that is used to iterate ,treverse, or retrive data form the collection or stream of object one by one.

Iterator
    Iterator is used Single directional curser.
    using iterator we can iterator the collection of objects in forward direction only.
        hasNext() -> returns true if there are more elements to iterate.
        next() -> returns the next element in the iteration.
        remove() -> removes the last element returned by the iterator.
ListIterator
    ListIterator is used for bidirectional cursor.
    using ListIterator we can iterate the collection of objects in both forward and backward direction.
        hasNext() -> returns true if there are more elements to iterate in the forward direction.
        next() -> returns the next element in the iteration in the forward direction.
        hasPrevious() -> returns true if there are more elements to iterate in the backward direction.
        previous() -> returns the previous element in the iteration in the backward direction.
        add() -> adds an element to the list at the current position of the iterator.
        set() -> replaces the last element returned by next() or previous() with the specified element.

Comparator 
    Comparator is a functional interface that is used to compare two objects of the same class.
    it is used to sort the collection of objects based on specific criteria.
    compare() 
        -1 -> first object is less than second object       asending order
        0 -> first object is equal to second object         equal
        1 -> first object is greater than second object     desending order
Comparable
    Comparable is a functional interface that is used to compare the current object with the specified object.
    it is used to sort the collection of objects based on natural ordering.
    compareTo()
        -1 -> current object is less than specified object      asending order
        0 -> current object is equal to specified object        equal
        1 -> current object is greater than specified object    desending order


                     Comparator                        Comparable
            used for custom sorting                  used for natural sorting
            compare() method is used to compare      compareTo() method is used to compare

Lambda Expression
    A lambda expression is a anonymous function that can be used to provide implementation of functional interface.
Functional Interface
    @FunctionalInterface is an annotation that used to indicate that the interface is a functional interface.
    An interface with only one abstract method , we can have any number of default or static method in functional interface.
Predicate
    Predicate is a functional interface with only one argument and return boolean value.
Stream
    Stream is a sequence of elements supporting sequential and parallel aggregate operations.
    Sequential Stream -> Process elements in order
    Parallel Stream -> Process elements in parallel using multiple threads.
    Flow
        Collection -> Stream -> Intermediate Operations -> Terminal Operations 
        Intermediate Operations -> filter(), map(), sorted()
        Terminal Operations -> forEach(), collect(), reduce()
optional Class
    Optional is a container object which may or may not contain a non-null value.
    it is used to avoid null pointer exception and provide better way to handle null values.
